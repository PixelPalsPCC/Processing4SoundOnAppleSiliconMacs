This is one of the most common headaches when moving Python code between different computers. The issue usually isn't that your file is in the wrong place, but that Python's "Current Working Directory" (CWD) changes depending on how or where you run the script.

If you run the script from a terminal or an IDE (like VS Code or PyCharm) that happens to be "looking" at your user home folder instead of your project folder, Python will look for your_audio.wav in your home folder and fail to find it.

To fix this permanently so that it works on any Mac without changes, you need to tell Python to look for the file relative to the script itself, rather than where the terminal is currently running.

The Fix: Use Dynamic Absolute Paths
Python has a built-in library called os that can detect exactly where your .py file is currently sitting on the hard drive.



Here is how to update your code to build the path dynamically:

Python
from p5 import *
from playsound3 import playsound
import threading
import os  # 1. Import the built-in operating system module

# 2. Get the exact folder path where THIS python file is saved
SCRIPT_DIR = os.path.dirname(os.path.abspath(__file__))

# 3. Combine that folder path with your audio file name
# (Change "your_audio.wav" to "data/your_audio.wav" if it's inside the data folder)
AUDIO_PATH = os.path.join(SCRIPT_DIR, "your_audio.wav")

def setup():
    size(400, 400)

def draw():
    background(240)

def play_audio_worker():
    # 4. Use the absolute path instead of just the filename
    playsound(AUDIO_PATH, block=True)

def mouse_pressed():
    audio_thread = threading.Thread(target=play_audio_worker)
    audio_thread.daemon = True 
    audio_thread.start()

if __name__ == '__main__':
    run()



    
Why this fixes the issue:
__file__ is a special Python variable that contains the full path to your script (e.g., /Users/george/Documents/Project/sketch.py).

os.path.dirname() strips away the filename, leaving just the folder path (/Users/george/Documents/Project/).

os.path.join() cleanly glues that folder path to your audio file name using the correct Mac forward slashes (/).

No matter what folder your IDE or terminal is open to on that other Mac, AUDIO_PATH will now point to the exact, absolute location of your sound file.
